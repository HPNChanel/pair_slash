# PairSlash Anti-Confusion Lines

Use these answers when someone immediately maps PairSlash onto a broader or
more familiar category.

## "Co phai day la agent framework nua khong?"

No. PairSlash is not a system for building or orchestrating more agents. It is
a narrow trust layer around a few high-value workflows people already repeat in
Codex CLI and GitHub Copilot CLI.

## "Khac gi bo prompt?"

A prompt bundle gives you reusable instructions. PairSlash gives you reusable
workflows with clearer trust boundaries: `/skills` entrypoints, repo grounding,
and explicit rules for what can become durable project truth.

## "Khac gi custom scripts?"

Custom scripts automate steps. PairSlash standardizes the parts that should be
reused while keeping judgment-heavy steps explicit. That matters when "almost
right" output still creates cleanup, trust debt, or bad durable memory.

## "Khac gi memory store?"

A memory store remembers things. PairSlash is about deciding what deserves
durable project truth and making that path disciplined. The point is not to
store more memory. The point is to keep the right memory trustworthy.

## "Tai sao chi 2 runtime?"

Because scope discipline is part of the product. PairSlash is built for the two
terminal-native runtimes that match the workflow model today: Codex CLI and
GitHub Copilot CLI. Adding a third runtime before the wedge is proven would
create fake breadth, not better trust.
