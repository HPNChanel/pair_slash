# PairSlash README Opening Options

Use these openings when the goal is fast comprehension and adoption, not
architecture signaling.

Recommended default: Version 1. It matches the strongest wedge order by leading
with repo re-entry pain before memory mechanics.

## Version 1: Hacker OSS

Coming back to a repo with terminal AI usually means paying the context tax
again: re-read files, sanity-check the model, and clean up almost-right output.
PairSlash is the trust layer for terminal-native AI workflows that gets you
back into a repo faster, makes repeated workflows safer to reuse, and keeps
project memory disciplined instead of letting chat history become fake truth.

It is not a new agent framework and not a black-box autopilot. PairSlash works
through `/skills` on exactly two runtimes: Codex CLI and GitHub Copilot CLI.

## Version 2: Maintainer / Team Lead

Your team does not need more AI enthusiasm. It needs less rework, fewer hidden
side effects, and a safer way to reuse what already works. PairSlash is the
trust layer for terminal-native AI workflows: it helps people re-enter repos
faster, run repeated workflows with clearer guardrails, and keep durable
project truth reviewable instead of scattering it across chats and private
notes.

It is not another framework for spawning more agents. It is a narrow,
`/skills`-first workflow layer for Codex CLI and GitHub Copilot CLI.

## Version 3: DevEx / Platform

Terminal AI adoption breaks down when every repo depends on private prompts,
fragile scripts, and trust-me memory. PairSlash is the trust layer for
terminal-native AI workflows: a practical way to help users understand repos
faster, reuse workflows more safely, and keep project memory disciplined
without hiding durable side effects.

It is not a black-box autopilot and not a third-runtime abstraction. PairSlash
standardizes high-trust workflows through `/skills` for Codex CLI and GitHub
Copilot CLI only.
