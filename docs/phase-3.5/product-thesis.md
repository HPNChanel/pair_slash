# PairSlash Product Thesis

## One-line thesis

PairSlash is the trust layer for terminal-native AI workflows that helps users understand repos fast, run repeatable workflows safely, and keep project memory disciplined across sessions and people.

## One-paragraph thesis

PairSlash exists to reduce the trust gap in terminal-native AI work. It helps users ground themselves in a repo faster, run repeated workflows without hidden side effects, and keep durable project memory explicit, reviewable, and reusable over time. Its center of gravity is disciplined memory and workflow trust, not more autonomy.

## Why now

- Terminal-native AI usage is moving from novelty to daily repo work. [Hypothesis]
- "Almost-right" AI output creates rework that users now feel often enough to care about a trust layer. [Hypothesis]
- Project context is still lost too easily across sessions, branches, and people. [Hypothesis]
- Teams will not scale terminal-native AI usage until they trust how durable truth is created and reused. [Hypothesis]

## Trust-layer framing

PairSlash is the layer between raw AI capability and durable project action.

It does three things:

1. makes repo understanding faster and less error-prone
2. makes repeated terminal workflows safer to reuse
3. makes project memory explicit, disciplined, and durable

Its operating posture is:

- authoritative-first project memory
- explicit-write-only durable changes
- `/skills` as the standard entrypoint

## Category definition

PairSlash is a trust layer for terminal-native AI workflows.

## Alternative categories PairSlash does not belong to

- a new agent framework
- an autonomous coding platform
- a general AI memory app
- a broad workflow automation suite
- a third-runtime abstraction layer
