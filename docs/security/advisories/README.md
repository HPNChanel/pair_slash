# Advisories

Store published PairSlash security advisories here when the repo has both:

- a concrete fix or mitigation
- evidence sufficient to describe the affected trust boundary honestly

Recommended file naming:

- `PSA-YYYY-001-short-title.md`

Each advisory should state:

- affected versions or commits
- affected runtime and target lanes
- impacted Phase 10 lane
- severity rationale
- remediation or patched release
- whether the claim is only internally tested, deterministic-lab backed, or live-evidence backed
