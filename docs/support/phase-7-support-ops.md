# PairSlash Phase 7 Support Ops

This page is the operational source of truth for PairSlash support bundles,
privacy boundaries, local-first telemetry summaries, and maintainer triage.

PairSlash support remains local-first:

- traces stay under `.pairslash/observability/`
- support bundles are generated only on explicit user command
- remote telemetry is off by default
- redaction runs before any bundle is marked shareable

## Support workflow

| Step | Actor | Input | Output | SLA / expectation |
| --- | --- | --- | --- | --- |
| Bug intake | Reporter | Live failure, runtime lane, command | Issue draft with session id and workflow context | Same session |
| Local debug | Reporter | Failed or latest session | `debug-report` with decisive failure domain and repro steps | Same session |
| Local export | Reporter | Session id or latest session | Redacted `trace-export` and optional `support-bundle` | Same session |
| Share gate | Reporter | Bundle manifest and privacy note | Shareable issue payload or local-only hold | Review before attaching |
| Triage | Maintainer | Issue plus bundle metadata | Failure-domain owner, severity, next action | 1 business day target |
| Reproduction | Maintainer | Bundle, doctor/context/policy artifacts | Reproducibility summary | 2 business day target |
| Decision | Maintainer | Repro summary plus patch/docs result | Fix, docs note, known issue, or close | Same issue cycle |

## Redaction policy

| Data class | Example | Store local? | Share in bundle? | Redaction action | Risk |
| --- | --- | --- | --- | --- | --- |
| Access secrets | API keys, PATs, private keys | No raw value | Never | Remove value entirely | Critical |
| Auth/session material | Cookies, bearer headers, OAuth tokens | No raw value | Never | Remove value entirely | Critical |
| Freeform sensitive text | Prompt bodies, memory statements, evidence text | Redacted only | Redacted only | Hash value | High |
| Runtime config content | MCP server config, env fragments, settings blobs | Fingerprint only | Fingerprint only | Replace with fingerprint | High |
| Personal absolute paths | Home paths, user-specific install roots | Normalized only | Normalized only | Replace with `<home>` or `<repo>` | Medium |
| Safe diagnostic metadata | Runtime, target, pack id, failure domain, reason codes | Yes | Yes | Keep as-is | Low |

Current redaction state values:

- `shareable`: known sensitive classes were removed or hashed and no unknown hit remained
- `review-required`: an unknown sensitive hit or unclear redaction case remains; do not share outside the machine

## Telemetry policy

Telemetry in Phase 7 is derived from local traces only. PairSlash does not
upload anything by default.

| Metric | Why needed | Local by default? | Remote opt-in? | Privacy note |
| --- | --- | --- | --- | --- |
| Workflow runs started | Basic workflow usage shape | Yes, via explicit local summary command | No default remote path | Derived from local trace indexes |
| Workflow runs succeeded | Detect usable completion rate | Yes, via explicit local summary command | No default remote path | Aggregate only |
| Weekly reuse days | Detect repeat use without identity tracking | Yes, via explicit local summary command | No default remote path | Counts days, not users |
| Median TTFS | Detect first-success friction | Yes, via explicit local summary command | No default remote path | Derived from session timeline only |
| Support bundle exports | Detect support load concentration | Yes, via explicit local summary command | No default remote path | Count only |

Use:

```bash
node packages/tools/cli/src/bin/pairslash.js telemetry summary --format text
node packages/tools/cli/src/bin/pairslash.js telemetry summary --format json --out artifacts/telemetry-summary.json
```

`telemetry summary` is an explicit local inspection step. It does not enable
background collection or remote upload.

## Support artifacts

| Artifact | Contains | Safe for sharing? | Best use case | Generated by |
| --- | --- | --- | --- | --- |
| `debug-report` | Decisive failure domain, timeline, repro steps | No by default | First local diagnosis | `pairslash debug` |
| `trace-export` | Redacted events, sessions index, redaction report | Conditional | Maintainer forensics | `pairslash trace export` |
| `support-bundle` | Trace export pointer, debug report, doctor/context/policy artifacts, privacy note, issue template | Conditional | Primary support attachment | `pairslash debug --bundle` or `pairslash trace export --support-bundle` |
| `reproducibility summary` | Live vs compat-lab repro decision | Yes | Maintainer close-out | Template in support bundle |
| `triage note` | Failure-domain routing and next action | Yes | Maintainer intake | Template in support bundle |

## Commands

Create a local debug report:

```bash
node packages/tools/cli/src/bin/pairslash.js debug --latest --runtime codex --format text
```

Create a support bundle from the latest session:

```bash
node packages/tools/cli/src/bin/pairslash.js debug --latest --runtime codex --bundle --format text
```

Create a support bundle from a selected session:

```bash
node packages/tools/cli/src/bin/pairslash.js trace export --session <session-id> --runtime codex --support-bundle --include-doctor --format text
```

## Rollout gate

| Maturity level | Required evidence | Blockers | Exit criteria |
| --- | --- | --- | --- |
| Usable alpha | Local debug, trace export, and support bundle commands work | No privacy note, no triage template, or no failure-domain summary | One full local rehearsal works |
| Support-ready | Privacy note, redaction state, issue template, maintainer templates, failure-domain routing are shipped | Unsafe share path or missing triage loop | Two rehearsals plus one real issue per primary runtime lane |
| Evidence-ready | Real issues processed end-to-end | Fake acceptance still mixed with live support | At least 4 real support cases, zero unsafe bundle shares |
| Release-ready | 30-day support evidence is stable | Privacy regressions or routing drift | At least 8 real cases across both runtimes with zero critical leak |
