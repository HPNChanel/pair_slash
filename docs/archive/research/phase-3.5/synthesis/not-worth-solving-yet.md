# PairSlash Phase 3.5 Not Worth Solving Yet

This file protects the roadmap from attractive but weakly evidenced requests.

## Exclusion Filters

Exclude or defer if the need is:

- low recurrence
- weakly evidenced
- poorly aligned to trust/context/repeatability
- broad but not wedge-shaping
- hard to measure

## Not Worth Productizing Yet

| Need that sounds attractive | Why it sounds good | Why it is not worth solving yet | What evidence would change the answer |
|---|---|---|---|
| Broad orchestration across many tools and runtimes | Feels strategic and ambitious | Dilutes trust-layer thesis and violates two-runtime discipline | Strong repeated demand tied to current ICP pain, without losing wedge focus |
| Black-box autopilot flows | Sounds powerful and demo-friendly | Conflicts with explicit-write-only and trust positioning | Evidence that users want less control and still trust durable outcomes |
| Generic prompt library or polished prompt pack | Easy to package and market | Weak moat, weak trust signal, easy to copy, not enough on its own | Repeated evidence that prompt packaging alone changes weekly behavior |
| Install polish as primary roadmap driver | Easy to quantify | Secondary unless core pain is already strong enough to pull adoption | Data showing strong recurring pain plus clear abandonment caused only by setup |
| Team-wide governance layer before local value is proven | Attractive to platform buyers | Risks solving governance theater before user pain | Segment data showing platform roles have both authority and urgent repeated pain |
| One-off consultant or OSS edge workflows with low repeat rate | Can generate interesting stories | Weak habit loop and poor wedge leverage | Multi-interview evidence that the same pattern repeats across repos and users |

## Rule

If a request does not strengthen a wedge around trust, context continuity, or
repeatable workflow value, keep it out of the next product round.
